# 競合調査の進め方

## 役割分担（このプラグインの思想）

- AI（あなた）: 候補出し・情報収集・裏取り・整理・比較・示唆出し
- 人（ユーザー）: どの競合を調べるかの**選択**と、結果を受けた**意思決定**

AI が正解を出すのではなく、**人が判断しやすい形に整える**のが仕事。

## 1. 候補の出し方

- 「**同じ課題を解決しているか**」を軸に選ぶ。見た目が違っても、顧客がそれで済ませられる
  なら競合（例: 家計簿アプリの競合には Excel テンプレートや銀行アプリも入りうる）。
- 直接競合（同じ形態・同じ客層）を中心に、**代替手段（間接競合）も1〜2個**混ぜると発見がある。
- 各候補は「名前＋一言説明＋なぜ競合か」で提示する。
- **実在を確認できないサービスを候補に入れない**（Web検索でヒットすることを確認する）。

## 2. 調査の観点（既定）

| 観点 | 見るもの |
|---|---|
| 料金 | 無料プランの範囲、有料プラン名と価格、課金単位（1ユーザーあたり月額 等）、年払い割引、無料トライアル |
| 主要機能 | 代表的な機能5〜8個。自社と重なる/重ならないが分かるように |
| 強み | そのサービスが選ばれている理由（確認できたもの） |
| 弱み | 事実ベースの中立表現で。「〜には対応していない」「〜は上位プランのみ」など |
| ターゲット | 誰向けか（個人/チーム/企業、業種、価格帯の客層） |
| 直近の動き | 値上げ・値下げ、新機能、日本展開など（分かる範囲で） |

## 3. 裏取りルール（最重要）

1. 信頼順: **公式サイト > 公式ヘルプ・公式ブログ > 大手メディア > 個人ブログ・比較サイト**。
2. 料金は**必ず公式の料金ページで確認**する。比較記事の数字をそのまま使わない（古いことが多い）。
3. 公式で確認できなかった数字は、書かないか「要確認」と明記して要確認事項に入れる。
4. 為替換算はしない（US$ 表記は US$ のまま書く）。参考換算を添える場合は
   「参考: 1ドル◯◯円で約◯◯円」と**換算であることを明示**する。
5. すべての出典に URL と確認日を残し、レポート末尾に出典一覧としてまとめる。

## 4. 価格の書き方

- 「月額980円（税込・1ユーザーあたり・年払い時）」のように、
  **金額・通貨・税の扱い・課金単位・支払いサイクル**をワンセットで書く。
- 月払いと年払いで価格が違う場合は、どちらの価格かを必ず書く（両方分かれば両方書く）。
- 税込か税別か公式に明記が無ければ「税表記なし」とする（勝手に決めない）。
- レポートの冒頭と料金比較表の直下に「**◯◯◯◯-◯◯-◯◯時点**」を明記する（料金は変わる前提）。

## 5. Webが使えない場合のフォールバック

- `materials/` の手元資料だけで作成し、レポート冒頭に「**手元資料のみで作成（Web未確認）**」と明記する。
- 手元資料も無ければ、正直に「このままでは調査できない」と伝える。
  ユーザーが「概要だけでも」と望んだ場合のみ一般知識で概要を書くが、
  **料金の具体的な数字は書かず**、冒頭に「Web未確認・概要のみ」と明記する。

## 6. トーン

- 中立・事実ベース。競合をけなさない、自社を過剰に持ち上げない。
- 「良い/悪い」ではなく「**誰に向いているか／自社とどう違うか**」で書く。
