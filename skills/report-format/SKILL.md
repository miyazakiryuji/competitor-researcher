# レポートの書式

## 1. Markdown レポート（必須成果物）

ファイル名: `reports/競合調査_{テーマ}_{YYYYMMDD}.md`
（テーマが長ければ短くする。例: `競合調査_料金と特徴_20260716.md`）

構成（この順で）:

1. **タイトル＋調査概要** — テーマ／実施日／対象（自社＋競合N社）／調査方法（Web調査、出典は末尾）／
   「**料金・仕様は ◯◯◯◯-◯◯-◯◯ 時点**」の断り書き
2. **結論サマリー** — 3〜5行。「忙しい人はここだけ読めばいい」が目標
3. **料金比較表** — 行=サービス（**自社を1行目**）、列=無料プラン／主な有料プラン／課金単位／備考。
   表の直下に時点表記
4. **特徴比較表** — 行=サービス、列=主要機能・強み・ターゲット（セルは短く。詳細は各社詳細へ）
5. **各社詳細** — 1社ごとに: 概要／料金プラン（箇条書き）／主要機能／強み／弱み／直近の動き
6. **自社との比較・ポジショニング** — 価格の位置・機能の位置・客層の重なりを簡潔に。
   2軸で整理すると分かりやすい場合は「縦軸=◯◯、横軸=◯◯」の言葉で説明する
7. **差別化のヒント（示唆）** — 「事実 → だから何が言えるか」をセットで3〜5個
8. **次のアクション案** — 明日からできる具体案を2〜3個（調査で終わらせない）
9. **出典一覧** — ページ名＋URL＋確認日
10. **要確認事項** — 確認できなかった数字・情報の一覧

書き方:

- 表のセルは短く。長い説明はセルに入れず、各社詳細に書く。
- 金額は「US$4/月（年払い・税表記なし）」のように単位・条件まで書く。
- 事実と意見を混ぜない。意見（解釈）は 6〜8 章にだけ書く。
- 自社の情報は `自社サービス.md` の記載に基づく。無い項目は「―」とする。

## 2. HTML ダッシュボード（既定で毎回生成）

ファイル名: Markdown と同名で拡張子 `.html`
（`リサーチの好み.md` が「つくらない」のとき、またはユーザーが不要と言ったときだけスキップ）

「ターミナルの文字列が、ブラウザで開ける1枚のダッシュボードになる」体験が売り。
表の羅列ではなく、**グラフで一目で分かる**構成にする。

### 構成（上から順に）

1. **ヘッダ** — テーマ・対象・「◯◯◯◯-◯◯-◯◯時点」のスタンプ
2. **結論サマリー** — 3〜4行（カード or 帯で目立たせる）
3. **料金グラフ（インラインSVGの横棒グラフ）**
   - **同一通貨（円建て）のプランだけ**を同じグラフに載せる。条件を揃える
     （例: すべて月払いの月額）。どの条件かをグラフ直下に明記する。
   - 外貨建てで比較できないサービスは、グラフの脇に注記カードで示す
     （例: 「TickTick: US$35.99/年（米ドル建てのため参考）」）。**為替換算して棒にしない**。
   - 自社の棒はアクセント色、競合はグレー系。棒の脇に金額を数字で書く。
4. **ポジショニングマップ（インラインSVGの2軸散布図）**
   - 軸はテーマに合わせて選ぶ（既定: 横軸=価格帯、縦軸=機能の多さ）。
   - 自社はアクセント色の●＋ラベル強調。
   - 位置づけは解釈を含むため、**「※配置は本調査に基づく解釈を含みます」と必ず注記**する。
5. **料金比較表** — Markdownレポートの表と同内容（自社行ハイライト）
6. **特徴比較表** — 同上
7. **差別化のヒント** — 番号付きで3〜5個
8. **出典** — フッターに小さく（詳細は .md 参照と書く）

### 技術要件

- **1ファイル自己完結**。外部 CSS/JS/画像/フォント/CDN を一切読み込まない
  （グラフは Chart.js 等を使わず、**インラインSVGで描く**）。ダブルクリックで開ける。
- **数字・内容は Markdown レポートと完全一致**させる。HTML化で情報を増やさない・変えない。
- A4横で1〜2枚に収まる印刷用スタイル（`@media print`）を入れる。
- スマホでも崩れない: 表は `<div style="overflow-x:auto">` のラッパで包む。SVG は `viewBox` ＋ `max-width:100%`。
- デザイン: 白背景＋アクセント1色（既定 #1f6feb 系）＋グレーの、カード型ダッシュボード調。
  `font-family: "Hiragino Sans", "Yu Gothic UI", Meiryo, sans-serif` 系。
  **自社はグラフ・表の両方でアクセント色ハイライト**して一目で分かるようにする。
- 生成後、ユーザーに「reports/ の .html をダブルクリックで開けます」と開き方を一言案内する。

## 3. PDF資料（ダッシュボードのPDF版・既定でつくる）

ファイル名: ダッシュボードと同名で拡張子 `.pdf`

- 生成方法: `scripts/html_to_pdf.py` にダッシュボードの `.html` を渡す
  （内部でヘッドレスの Chrome / Edge / Chromium を自動検出して `--print-to-pdf` する）。
- **内容はダッシュボードの印刷そのもの**。PDF用に情報を増やしたり別レイアウトを作ったりしない
  （見た目の一貫性と数字の一致を保つため。レイアウトは `@media print` / `@page` 側で調整する）。
- ブラウザが見つからない環境ではスクリプトが終了コード 2 で案内を出すので、
  「.html を開いて 印刷 → PDFとして保存」の手動手順を伝えてスキップする（エラー扱いにしない）。
- 用途の目安を一言添える: 印刷して会議に持ち込む・PDFしか添付できない場面用。
  画面共有・チャット共有には `.html` のほうがきれい。
