# 専属リサーチャー（competitor-researcher）

## 概要

自社（または自分の）サービスのプロフィールを一度登録するだけで、競合サービスの
**料金と特徴**を Web で調査し、**出典付きの比較レポート**まで自動で作る Claude Code プラグインです。

- `自社サービス.md` を書く → `/competitor-researcher:research` だけで、
  `reports/競合調査_{テーマ}_{日付}.md` と、**グラフ入りのHTMLダッシュボード**（料金の棒グラフ＋
  ポジショニングマップ入り・1ファイルでブラウザに開ける）、さらにその**PDF資料**まで毎回できます。
- 競合の**候補出しは AI**、**どれを調べるかはあなた**が選びます。
- 料金の数字は公式サイトで裏取りし、**通貨・税・課金単位・確認日**を必ず明記します。
  確認できなかったことは捏造せず「要確認事項」にまとめます。
- 時間をおいて再実行すれば、最新の料金で作り直せます（定点観測）。

「専属」の意味: `自社サービス.md` と `リサーチの好み.md` を毎回読み込むので、
2回目からは何も説明しなくても、**自社目線での比較・示唆**まで出してくれます。

## 前提

| 用途 | 必要なもの | 無い場合 |
|---|---|---|
| プラグイン実行 | Claude Code（最新版・有料プラン） | — |
| 料金・特徴の調査 | Web 検索が使えること | `materials/` に手元資料を置けば、それだけでレポートを作れます |
| PDF資料の自動生成 | Google Chrome または Microsoft Edge | 自動でスキップされます（`.html` をブラウザで開き「印刷 → PDFとして保存」で同じものが作れます） |

追加のライブラリ・外部ツールは**不要**です（同梱スクリプトは Python 標準ライブラリのみ）。

## 導入（コピペのみ・GitHub アカウント不要）

取り込む人に GitHub アカウントは**不要**です（マニフェストの author/owner 名は表示用ラベルで、
認証には使いません）。次の A〜C のどれか1つで導入できます。

**A. GitHub から（おすすめ・コピペ2行）**
```
/plugin marketplace add miyazakiryuji/competitor-researcher
/plugin install competitor-researcher@competitor-researcher
```
※ 一覧に出てこない場合だけ `/plugin enable competitor-researcher@competitor-researcher`

**B. 配布フォルダから（恒久インストール）**
```
/plugin marketplace add "<competitor-researcher フォルダのパス>"
/plugin install competitor-researcher@competitor-researcher
```
※ zip で受け取った場合は、**先にダブルクリック等で解凍**し、できたフォルダのパスを指定してください
（`/plugin marketplace add` は zip を直接指定できません）。

**C. いちばん手軽（そのセッションだけ有効）**
```bash
claude --plugin-dir "<competitor-researcher フォルダ か .zip>"
```
※ こちらは zip のままでも使えます。

配布用 zip は `bash scripts/make_dist.sh` で作れます。

### 更新したいとき

いったんアンインストールして、入れ直すのが確実です:
```
/plugin uninstall competitor-researcher@competitor-researcher
/plugin install competitor-researcher@competitor-researcher
```

## 使い方（3ステップ）

1. **`/competitor-researcher:init`** — 作業フォルダ一式を作ります。
2. **`自社サービス.md` を記入** — 分かるところだけでOK。
   すぐ試したい場合は `サンプル_自社サービス.md`（架空のタスク管理アプリ「タスカル」）をコピー。
3. **`/competitor-researcher:research`** — 競合候補が5〜7個提案されるので、番号で選ぶと調査開始。

### コマンド一覧

| コマンド | 何をする |
|---|---|
| `/competitor-researcher:init` | 作業フォルダ一式を作成（既存は上書きしない） |
| `/competitor-researcher:research` | 競合候補の提案 → 調査 → 比較レポート生成 |
| `/competitor-researcher:research Todoist TickTick` | 競合を指定して調査（候補提案をスキップ） |
| `/competitor-researcher:research 料金プランの見直し` | テーマを指定して調査 |

### 作業フォルダの構成

```
（任意のフォルダ）/
├── はじめにお読みください.md
├── 自社サービス.md          # あなたのサービスのプロフィール（毎回読み込み）
├── サンプル_自社サービス.md  # 架空サービスの記入例（すぐ試せる）
├── リサーチの好み.md        # 競合数・観点・HTMLシート要否などの既定値
├── materials/               # 手元資料置き場（任意）
└── reports/                 # 生成された調査レポート（.md / .html）
```

## レポートに含まれるもの

**Markdown レポート（.md）**

1. 調査概要（テーマ・日付・対象・「◯◯時点」の断り）
2. 結論サマリー（3〜5行）
3. 料金比較表（自社を1行目に）
4. 特徴比較表
5. 各社詳細（料金プラン・主要機能・強み・弱み・直近の動き）
6. 自社との比較・ポジショニング
7. 差別化のヒント（示唆）
8. 次のアクション案
9. 出典一覧（URL＋確認日）
10. 要確認事項（確認できなかったこと）

**HTML ダッシュボード（.html・既定で毎回生成）**

- 結論サマリー／料金の横棒グラフ／ポジショニングマップ（2軸）／料金・特徴比較表／差別化のヒント
- グラフはインラインSVG描画で**1ファイル自己完結**（外部読み込みなし）。ダブルクリックで開けて、
  A4横の印刷にも対応。数字は .md レポートと完全一致します。

**PDF 資料（.pdf・既定でつくる）**

- ダッシュボードを A4横 でそのままPDF化したもの（Chrome / Edge を自動検出して変換）。
- 印刷して会議に持ち込む・PDFしか添付できない場面用。画面共有には .html のほうがきれいです。

## よくある質問

**Q. レポートの料金が実際と違う気がする**
料金は頻繁に変わります。レポートには確認日が明記されているので、まず日付を確認してください。
もう一度 `/competitor-researcher:research` を実行すれば、最新の情報で作り直せます。
それでも合わない場合は出典URLを直接確認してください（レポート末尾にあります）。

**Q. 会社のネットワークで Web 検索が使えない**
`materials/` に手元資料（料金表のメモなど）を置けば、それだけでレポートを作れます。
その場合、レポート冒頭に「手元資料のみで作成」と明記されます。

**Q. 競合の候補が的外れだった**
`自社サービス.md` の「一言でいうと」「ターゲット」を具体的にすると精度が上がります。
また「意識している競合」欄に書いた名前は必ず候補に入ります。

**Q. HTML ダッシュボードが要らない／毎回確認してほしい**
`リサーチの好み.md` の「HTMLダッシュボード」を「つくらない」または「毎回きく」に書き換えてください
（既定は「毎回つくる」です）。PDF も同様に「PDF資料」の項目で変えられます。

**Q. PDF ができなかった**
Chrome / Edge が見つからない環境では PDF 生成は自動スキップされます。
`.html` をブラウザで開いて 印刷（Cmd/Ctrl+P）→「PDFとして保存」で、同じ内容のPDFが作れます。

**Q. 実在の会社を調査していいの？**
公開されている料金・機能を調べて比較する通常の競合調査であり、問題ありません。
レポートは中立な事実ベースの文体で書かれるため、そのまま社内共有できます。
