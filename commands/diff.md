# 定点観測（diff）— 前回からの変化レポート

あなたは `competitor-researcher` プラグインの定点観測担当です。**すべて日本語**で、
過去2回分の調査を突き合わせ、「前回から何が変わったか」だけを1枚にまとめます。

## 必ず守るルール

- 変化の判定は**両方のレポートに書かれている内容だけ**から行う。記憶や推測で補わない。
- 「変わっていない」ことも重要な結果として明記する（値上げなし＝安心材料）。
- 新旧の数字には、それぞれのレポートの**時点（YYYY-MM-DD）**を必ず添える。

作業ルートは現在の作業ディレクトリ（`reports/` があるフォルダ）。以下 `WS` と呼ぶ。

## Step 1. 比較対象を特定する

1. `WS/reports/_data/` に JSON（機械可読データ）があれば、それを優先して使う。
   無ければ `WS/reports/` の Markdown レポートを使う。
   **比較対象は `競合調査_` で始まるレポート（と `_data/` の JSON）のみ**。
   `変化レポート_` で始まるファイル（この diff 自身の過去の出力）は対象にしない。
2. `$ARGUMENTS` で日付やファイル名が指定されていればそれに従う。
   指定がなければ、**同じテーマの最新2件**（日付の新しい順）を選ぶ。
3. 対象が1件以下しか無い場合は中止し、こう伝える:
   > 比較には2回分の調査が必要です。まず `/competitor-researcher:research` を実行して、
   > 時間をおいて（例: 来月）もう一度実行してから、再度お試しください。
4. 比較する2件（日付とファイル名）をユーザーに1行で示してから進める。

## Step 2. 突き合わせる

新旧それぞれについて、**サービスごと・プランごと**に対照する:

- **料金の変化**: プラン名／旧価格（旧時点）→ 新価格（新時点）。増減額・増減率も添える。
  片方にしか無いプラン（新設・廃止）は「新設」「記載なし（廃止または名称変更の可能性）」と書く。
- **無料プランの変化**: 制限の増減。
- **機能・直近の動き**: 新しく登場した機能・発表。
- **調査対象のずれ**: 新旧で調べた会社が違う場合は、比較できるのは共通する会社だけであることを明記する。

## Step 3. 変化レポートを作成する

`WS/reports/変化レポート_{テーマ}_{新しい方の日付}.md` に保存する。構成:

1. **概要** — 比較した2時点・対象サービス
2. **変化サマリー**（3行以内）— 例: 「Todoist が値上げ。他2社は変化なし」
3. **料金の変化表** — 行=サービス×プラン、列=旧価格（時点）／新価格（時点）／変化
4. **機能・動きの変化** — 箇条書き
5. **示唆** — 変化を受けて自社が検討すべきこと（2〜3個。変化なしなら「静観でよい」と書いてよい）
6. **注記** — 「各時点の数字は当時の出典に基づく。最新の確定情報は新しい方のレポートの出典を参照」

## Step 4. 締め

生成したファイルのパスと、変化サマリーの1行を示して終わる。
値上げ・値下げが見つかった場合は「該当社の出典URL（新レポート記載）を直接確認することをおすすめします」と添える。
