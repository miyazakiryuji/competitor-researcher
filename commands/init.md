# 専属リサーチャーの初期化（init）

あなたは `competitor-researcher` プラグインの初期化担当です。以下を順番に、日本語で実行してください。

## 1. 作成先フォルダを決める

- `$ARGUMENTS` が与えられていれば、それを作成先フォルダとして使う。
- 与えられていなければ、現在の作業ディレクトリ（`pwd`）を作成先とする。
- 現在のフォルダに既にプロジェクト（大量の無関係なファイルや `.git`）があって、そこに作業
  フォルダを足すのが不自然に見える場合は、作成先をユーザーに1回だけ確認する。
  それ以外は確認せずに進めてよい（`init_workspace.py` は既存を上書きしないので安全）。

## 2. 作業フォルダを作成する

Bash ツールで次を実行する（作成先が引数のときは末尾にパスを付ける）:

```bash
python3 "${CLAUDE_PLUGIN_ROOT}/scripts/init_workspace.py" [作成先フォルダ]
```

このスクリプトは以下を行う（**既存のフォルダ・ファイルは上書きしない**。何度実行しても安全）:

- フォルダ作成: `materials/`（手元資料置き場）, `reports/`（レポート保存先）,
  `reports/_data/`（diff 用の機械可読データ置き場）
- テンプレート作成: `はじめにお読みください.md`、`自社サービス.md`（記入用テンプレート）、
  `サンプル_自社サービス.md`（架空サービスの記入例）、`リサーチの好み.md`、`materials/使い方.md`

スクリプトの出力（作成した／スキップした一覧）をそのままユーザーに見せる。

## 3. 次の一歩を案内する

最後に、以下だけを短く案内して終わる:

1. `自社サービス.md` を開いて、**分かるところだけ**記入する
   （すぐ試したい場合は `サンプル_自社サービス.md` の内容をそのままコピーすればOK、と添える）
2. 記入できたら `/competitor-researcher:research` を実行する

ユーザーに頼まれない限り、ここで止まる（勝手に調査を始めない）。
